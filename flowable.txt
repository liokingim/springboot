
Flowable 7은 배포 방식이 예전 6.x와 달라진 부분이 있어서, CentOS 9 기준으로 “직접 실행 가능한 순서”와 “Tomcat/WAR 대체 방식”을 나눠 정리하겠습니다. 공식 문서 기준으로 버전·요구사항도 함께 확인해서 맞추겠습니다.

28s 동안 생각함
아래 순서로 설치하는 것을 추천합니다.
Flowable 7.x는 예전처럼 Flowable UI WAR를 Tomcat에 올리는 방식이 아니라, Spring Boot 3 기반 애플리케이션에 Flowable 엔진/REST를 포함해서 운영하는 방식으로 보는 것이 좋습니다. Flowable 7.0.0은 Java 17, Spring Boot 3, Spring 6 기반으로 전환되었고, UI 애플리케이션은 Flowable 7 OSS에서 제외되었습니다. 

1. 설치 방식 결정
CentOS 9에서는 다음 구성을 추천합니다.

CentOS Stream 9
 ├─ Java 17
 ├─ Maven
 ├─ MySQL 8 또는 MariaDB 10.6+
 └─ Spring Boot 3 + Flowable 7.2.0
Flowable 7 계열 중 Maven Repository 기준 7.x 최신은 7.2.0입니다. 8.0.0도 존재하지만, 질문이 Flowable 7이므로 7.2.0 기준으로 작성합니다. 

2. CentOS 9 기본 패키지 업데이트
sudo dnf update -y
sudo dnf install -y wget curl unzip git vim
서버 시간대가 필요하면 다음처럼 설정합니다.

sudo timedatectl set-timezone Asia/Seoul
timedatectl
3. Java 17 설치
Flowable 7은 Java 17 기반으로 보는 것이 안전합니다. Flowable 7.0.0 릴리스 설명에서도 Java 17, Spring Boot 3, Spring 6 전환이 명시되어 있습니다. 

sudo dnf install -y java-17-openjdk java-17-openjdk-devel
확인:

java -version
javac -version
예상:

openjdk version "17..."
javac 17...
환경변수 설정:

readlink -f $(which java)
보통 다음 경로와 비슷합니다.

/usr/lib/jvm/java-17-openjdk-...
/etc/profile.d/java.sh 생성:

sudo vi /etc/profile.d/java.sh
내용:

export JAVA_HOME=/usr/lib/jvm/java-17-openjdk
export PATH=$JAVA_HOME/bin:$PATH
적용:

source /etc/profile.d/java.sh
echo $JAVA_HOME
4. Maven 설치
sudo dnf install -y maven
확인:

mvn -version
5. DB 설치
Flowable은 관계형 DB를 사용합니다. 공식 요구사항에는 MySQL 8.0, MariaDB 10.6+ 등이 지원 DB로 정리되어 있습니다. 

CentOS 9에서 가볍게 운영한다면 MariaDB가 편합니다.

sudo dnf install -y mariadb-server mariadb
sudo systemctl enable --now mariadb
sudo systemctl status mariadb
보안 설정:

sudo mysql_secure_installation
DB와 계정 생성:

sudo mysql -u root -p
CREATE DATABASE flowable DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'flowable'@'localhost' IDENTIFIED BY 'flowable_password';

GRANT ALL PRIVILEGES ON flowable.* TO 'flowable'@'localhost';

FLUSH PRIVILEGES;
EXIT;
외부 WAS나 다른 서버에서 접근할 경우:

CREATE USER 'flowable'@'%' IDENTIFIED BY 'flowable_password';
GRANT ALL PRIVILEGES ON flowable.* TO 'flowable'@'%';
FLUSH PRIVILEGES;
6. Flowable Spring Boot 프로젝트 생성
작업 디렉터리:

mkdir -p ~/apps
cd ~/apps
Spring Boot 프로젝트 생성:

mkdir flowable7-app
cd flowable7-app
pom.xml 생성:

vi pom.xml
내용:

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="
         http://maven.apache.org/POM/4.0.0
         https://maven.apache.org/xsd/maven-4.0.0.xsd">

    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>flowable7-app</artifactId>
    <version>1.0.0</version>
    <packaging>jar</packaging>

    <properties>
        <java.version>17</java.version>
        <spring.boot.version>3.5.4</spring.boot.version>
        <flowable.version>7.2.0</flowable.version>
    </properties>

    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-dependencies</artifactId>
                <version>${spring.boot.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>

            <dependency>
                <groupId>org.flowable</groupId>
                <artifactId>flowable-bom</artifactId>
                <version>${flowable.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>

    <dependencies>
        <!-- Web API -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- Flowable BPMN Process Engine -->
        <dependency>
            <groupId>org.flowable</groupId>
            <artifactId>flowable-spring-boot-starter-process</artifactId>
        </dependency>

        <!-- Flowable REST API -->
        <dependency>
            <groupId>org.flowable</groupId>
            <artifactId>flowable-spring-boot-starter-process-rest</artifactId>
        </dependency>

        <!-- DB -->
        <dependency>
            <groupId>org.mariadb.jdbc</groupId>
            <artifactId>mariadb-java-client</artifactId>
        </dependency>

        <!-- Health Check -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-actuator</artifactId>
        </dependency>

        <!-- Test -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
                <version>${spring.boot.version}</version>
            </plugin>
        </plugins>
    </build>

</project>
Flowable 문서에서도 Spring Boot 프로젝트에 flowable-spring-boot-starter 또는 관련 starter를 추가하는 방식으로 시작한다고 설명합니다. 

REST만 필요하면 flowable-spring-boot-starter-process-rest를 사용하는 방식이 맞습니다. Maven Central 설명도 이 모듈을 “Flowable Process engine REST API standalone” starter로 설명합니다.

8. application.yml 설정
mkdir -p src/main/resources
vi src/main/resources/application.yml
내용:

server:
  port: 8080

spring:
  datasource:
    url: jdbc:mariadb://localhost:3306/flowable?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Seoul
    username: flowable
    password: flowable_password
    driver-class-name: org.mariadb.jdbc.Driver

  jackson:
    time-zone: Asia/Seoul

flowable:
  database-schema-update: true
  async-executor-activate: true
  check-process-definitions: true

management:
  endpoints:
    web:
      exposure:
        include: health,info
처음 실행할 때 Flowable 테이블을 자동 생성하려면 database-schema-update: true가 필요합니다. Flowable 공식 요구사항에서도 초기 실행 시 스키마 자동 생성을 위해 DB DDL 권한이 필요하다고 설명합니다. 

9. 테스트용 BPMN 파일 추가
mkdir -p src/main/resources/processes
vi src/main/resources/processes/simple-approval.bpmn20.xml
내용:

<?xml version="1.0" encoding="UTF-8"?>
<definitions xmlns="http://www.omg.org/spec/BPMN/20100524/MODEL"
             xmlns:flowable="http://flowable.org/bpmn"
             targetNamespace="Examples">

    <process id="simpleApproval" name="Simple Approval" isExecutable="true">

        <startEvent id="startEvent" />

        <sequenceFlow id="flow1" sourceRef="startEvent" targetRef="teamLeaderApproval" />

        <userTask id="teamLeaderApproval"
                  name="팀장 결재"
                  flowable:candidateGroups="TEAM_LEADER" />

        <sequenceFlow id="flow2" sourceRef="teamLeaderApproval" targetRef="departmentHeadApproval" />

        <userTask id="departmentHeadApproval"
                  name="부서장 결재"
                  flowable:candidateGroups="DEPARTMENT_HEAD" />

        <sequenceFlow id="flow3" sourceRef="departmentHeadApproval" targetRef="endEvent" />

        <endEvent id="endEvent" />

    </process>
</definitions>
이 파일은 다음 흐름입니다.

기안 시작 → 팀장 결재 → 부서장 결재 → 종료

12. systemd 서비스 등록
운영용으로는 직접 java -jar로 실행하지 말고 systemd에 등록하는 것이 좋습니다.

sudo mkdir -p /opt/flowable7
sudo cp target/flowable7-app-1.0.0.jar /opt/flowable7/flowable7-app.jar
실행 계정 생성:

sudo useradd -r -s /sbin/nologin flowable
sudo chown -R flowable:flowable /opt/flowable7
서비스 파일 생성:

sudo vi /etc/systemd/system/flowable7.service
내용:

[Unit]
Description=Flowable 7 Spring Boot Application
After=network.target mariadb.service

[Service]
User=flowable
Group=flowable
WorkingDirectory=/opt/flowable7
ExecStart=/usr/bin/java -jar /opt/flowable7/flowable7-app.jar
SuccessExitStatus=143
Restart=always
RestartSec=10

Environment=JAVA_HOME=/usr/lib/jvm/java-17-openjdk
Environment=SPRING_PROFILES_ACTIVE=prod

[Install]
WantedBy=multi-user.target
등록 및 실행:

sudo systemctl daemon-reload
sudo systemctl enable --now flowable7
sudo systemctl status flowable7
로그 확인:

journalctl -u flowable7 -f

14. Nginx 리버스 프록시 선택 사항
운영에서는 8080을 직접 열기보다 Nginx를 앞에 두는 것이 좋습니다.

sudo dnf install -y nginx
sudo systemctl enable --now nginx
설정 파일:

sudo vi /etc/nginx/conf.d/flowable7.conf
내용:

server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8080;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
검사 및 재시작:

sudo nginx -t
sudo systemctl restart nginx
방화벽:

sudo firewall-cmd --add-service=http --permanent
sudo firewall-cmd --reload
15. Tomcat에 올리고 싶을 때 주의점
Flowable 7 REST는 Jakarta 9 이상을 지원하는 Servlet Container가 필요합니다. Flowable 7.0.0 릴리스 노트에도 REST 애플리케이션은 Jakarta 9 지원 서버가 필요하다고 되어 있습니다. 

따라서:

Tomcat 9  → 비추천 / javax 기반
Tomcat 10 → 가능 / jakarta 기반
Tomcat 10.1 → 추천
하지만 지금 구성에서는 Spring Boot 내장 Tomcat 10.x로 JAR 실행하는 것이 가장 단순합니다. 외부 Tomcat WAR 배포는 굳이 필요하지 않습니다.

16. Flowable 7에서 UI가 없는 문제
Flowable 7 OSS는 핵심 엔진과 REST API 중심으로 바뀌었고, 7.0.0 릴리스 노트에 UI applications, content, form engines 등이 제외되었다고 명시되어 있습니다. 

따라서 업무 결재 시스템을 만들 경우 추천 구조는 다음입니다.

[사용자 화면]
Vue / React / Thymeleaf / JSP 등 직접 구현

        ↓ REST API 호출

[업무 서버]
Spring Boot + Flowable 7

        ↓

[DB]
MariaDB / MySQL
결재 화면, 결재선 관리, 반려 후 재상신, 조직별 문서 관리자 설정 등은 Flowable UI에 의존하지 말고 직접 테이블과 화면을 만드는 것이 좋습니다.

17. 업무 결재용 추천 테이블
Flowable 테이블과 별도로 업무 시스템 테이블을 두는 것이 좋습니다.

-- 사용자
APP_USER

-- 조직
APP_ORG

-- 직책
APP_POSITION

-- 문서 종류
APP_DOC_TYPE

-- 문서 종류별 관리자
APP_DOC_MANAGER

-- 결재 문서
APP_APPROVAL_DOC

-- 결재선
APP_APPROVAL_LINE

-- 결재 이력
APP_APPROVAL_HISTORY

-- Flowable 프로세스 연결
APP_PROCESS_LINK
APP_PROCESS_LINK 예시:

CREATE TABLE APP_PROCESS_LINK (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    approval_doc_id BIGINT NOT NULL,
    process_instance_id VARCHAR(128) NOT NULL,
    process_definition_key VARCHAR(128) NOT NULL,
    current_status VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
